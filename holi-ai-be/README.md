# HoliAI - Backend

This is the Fastify and BullMQ backend worker for **HoliAI**. It securely manages API keys, parses dynamic LLM models, and powers the **Autonomous Self-Evolving AI Persona**.

### Key Backend Capabilities
- **Autonomous Tool-Calling & Modular Worker**: Implements a robust multi-turn execution loop allowing the LLM to autonomously trigger database tools (fetching plans, memories, and crons) only when context is strictly required. The worker layer is strictly decoupled into single-responsibility modules (`agentRunner`, `errorHandler`, `toolHandlers`, etc.) to enforce high maintainability.
- **Local RAG & Embeddings**: Uses the raw ONNX runtime to locally embed conversations via `all-MiniLM-L6-v2`, performing highly-accurate semantic searches via `pgvector` in PostgreSQL.
- **Language Cognition Directives**: The backend router dynamically constructs absolute LLM constraints (`langRule`) before appending messages to the Redis Queue, forcing the AI to strictly output and reason in either English or Czech based on frontend UI state.
- **Background Autonomy**: Built-in NodeJS cron workers process offline tasks, background routines, and dynamic alarms.
- **Decoupled Database Architecture**: The data layer is decoupled into strictly separated domains (`crons.js`, `facts.js`, `episodic.js`, `biometrics.js`, `actionModules.js`) guaranteeing strict separation of concerns over the PostgreSQL schema engine.
- **Question Queue Context Engineering & RAG Purity**: Eliminates prompt bloat by enforcing strict Question Queue-based context retrieval. The AI manages a queue of pending questions, which triggers backend background RAG to fetch semantic memory and user facts seamlessly *one question at a time*. The queue guarantees strict **RAG Semantic Purity** (default relevance threshold `0.5`) by retrieving only relevant facts without injecting raw chat history. The internal Queue Router automatically detects and skips obsolete questions, instantly dequeuing and fetching RAG context for the next valid question without additional prompt overhead. Features deep payload minification to massively shrink redundant tool schema data before LLM execution.
- **Robust Retry Orchestration**: Built-in orchestration loops ensure transparent exponential backoffs on `500` and `503` server errors, while gracefully handling `429 Too Many Requests` and intermediate AI reasoning steps by logging system messages directly into episodic memory for a fully transparent chat history.
- **Resilient Tool Hallucination Recovery**: Incorporates a highly robust, fault-tolerant JSON extraction engine (`safeJsonParse`) that intercepts, sanitizes, and successfully parses malformed payloads or stringified tool arguments returned by open-source LLMs. Includes automated regex-based recovery for undocumented API schema rejections (like Groq `failed_generation` errors) to immediately self-heal the orchestrator without triggering unnecessary network retry loops or test suite timeouts.
- **Redis Write-Through Caching**: A decoupled caching layer that securely persists backend configurations and eliminates Supabase/Neon database query bottlenecks.
- **Token Optimization Architecture**: Completely eliminates token bloat by extracting and injecting ONLY the very last assistant message from the chat history during the active tool-calling loop. Historical context is derived exclusively from the Question Queue semantic RAG, meaning continuous multi-turn conversations operate at O(1) context cost.
- **LLM Trace Logging**: Deeply captures and records token consumption, latencies, and raw input/output JSON payloads into a unified database structure (`llm_traces`), enabling detailed frontend visualization and developer telemetry.
- **Robust Test Coverage & Verification**: Implements strict 80% global and branch coverage thresholds powered by Jest, rigorously testing the `errorHandler`, `agentRunner`, and complex conditional orchestration logic to guarantee high reliability of the background worker.
- **Complete Environment Resetting**: Implements safe, cascading database wiping architecture allowing users to easily nuke all their records, traces, and queued background tasks for a completely fresh start.
